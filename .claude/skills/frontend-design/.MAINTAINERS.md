# Maintaining the HushBox frontend-design skill

This skill is a merge of four upstream sources, adapted to HushBox. It runs a
generate -> detect -> audit -> adjudicate -> fix loop. This file records what came
from where, what we changed, and how to keep it current.

## Architecture

The loop, the invariants (detector on the main agent; one audit subagent, blind to the detector
until adjudication; direction-aware grading against `DESIGN.md`; the no-cap stop gate), and the
two standing notes are defined in `SKILL.md`; the diagram is in `.flow.md`. This file does not
restate them. Two facts worth pinning here: there is exactly one review subagent
(`.claude/agents/design-review.md`, no Codex agent and no second reviewer), and it is OneRedOak's
reviewer made direction-aware, absorbing the technical audit (a11y, perf, theming, responsive,
anti-patterns).

## HushBox tuning (decisions baked in)

- **Expressive by default.** Dial baseline `VARIANCE 8 / MOTION 6 / DENSITY 4` (taste-skill
  default). Reduced / no motion is opt-in through the accessibility widget, not the default.
  Per-surface tuning pulls dense product surfaces (settings, billing) down from VARIANCE 8.
- **Fonts (canonical in DESIGN.md):** Merriweather for reading (chat messages, marketing prose),
  Hanken Grotesk for product UI chrome (buttons, labels, data, settings), JetBrains Mono for code.
  Hanken Grotesk was chosen as a warm humanist sans, legible at small sizes and not on the
  overused list. Avoid Inter / Geist / Plus Jakarta Sans / Space Grotesk / DM Sans.
- **Committed != reflex.** A common-looking choice that is deliberate and reasoned is not slop.
  Rarity is not the test, intent is. This is why `cream-palette` is suppressed (see below).
- **No long dashes in user-facing copy.** Stated once, in `docs/DESIGN.md`. Internal text (these notes, the reference guides, code comments) is exempt.
- **Committed identity** (warm paper, the serif voice, the single brand red plus a distinct error
  red, 8px radius, warm-charcoal dark) is canonical in `docs/DESIGN.md`; the strategic principles
  are in `docs/PRODUCT.md`. SKILL.md points to both rather than restating them.

## Detector suppression and pins

- `.impeccable/config.json` sets `detector.ignoreRules: ["cream-palette"]` so the detector
  stops flagging HushBox's committed warm-paper background. `detector.designSystem.enabled`
  stays true so `design-system-font/color/radius` enforce DESIGN.md tokens.
- The detector requires the four static-HTML deps: `htmlparser2`, `css-select`, `css-tree`,
  `domutils` (added to the repo). The upstream update-check (the daily version poll to
  impeccable.style, gated by `IMPECCABLE_NO_UPDATE_CHECK`) was removed; runs are always hermetic.
- The detector declared `engines.node: ">=24"`, but runs on the repo's Node 22: the APIs it uses
  (AbortSignal.timeout, fetch, top-level await) all exist in 22, so the floor was conservative.
  No Node bump was required.
- `PRODUCT.md` and `DESIGN.md` live in `docs/`; the sidecar `.impeccable/design.json` lives with
  `config.json`. Both `context.mjs` and the detector resolve `docs/` natively (their vendored
  search includes it), so no code edit is needed. Keep these files only in `docs/` so a stray copy
  at root cannot shadow them.
- Exit codes (load-bearing for the loop): `0` clean, `2` findings, `1` detector missing.
  Finding schema: `{antipattern, name, description, severity, file, line, snippet}`.
  Keep the `scripts/` directory layout intact; `detect.mjs` resolves `detector/` and `lib/`
  by relative path.

## Disagreement log (where we override an upstream directive, and why)

1. "Common colors are slop" (`cream-palette`, taste-skill palette ban, Anthropic slop cluster
   #1) -> overridden. Warm off-white is optimal for a reading surface. Suppress the rule, record
   committed identity, encode "committed != reflex".
2. "Serif is the #1 AI tell" (taste-skill) -> overridden at the brand level. Merriweather is
   committed identity. Product UI chrome still moves to a sans (Hanken Grotesk) per the product
   register.
3. taste-skill motion-maximalism -> ADOPTED by HushBox choice (expressive by default), with
   reduced motion as an a11y opt-in.
4. Em-dash rules conflict between sources -> resolved to: no long dashes in user-facing copy (internal text, including these notes and the reference guides, is exempt). Stated once, in `docs/DESIGN.md`.
5. "Must generate imagery" (taste-skill) -> scoped to marketing only; product UI needs none.
6. "Single-font = slop" vs product register "one family" -> product uses a sans for chrome plus
   serif for reading plus mono for code; `single-font` slop rule suppressed in principle.
7. taste-skill Next.js / Motion-mandatory stack -> HushBox is Vite + React 19; motion uses both
   Framer Motion (orchestration) and CSS keyframes + View Transitions + useAnimationFrame (cheap
   and native), split by role; all gated by the a11y layer.
8. Plan's "exit 0 AND review PASS, cap 3" -> replaced by the no-cap adjudication gate above.
9. Impeccable critique's Codex sub-agent permission choreography -> dropped; Claude Code spawns
   the one native subagent directly. Snapshot/trend persistence dropped (loop is in-memory).
10. `onboard.md` reclassified as a refine guide, not bootstrap.

## Maintenance (this is now a HushBox-owned fork)

The skill has intentionally diverged from its upstreams and is maintained directly. `vendor.mjs`
and the re-vendor flow were removed; the dead URL/browser path, the live-DOM `*DOM` adapters, and
the profiler were deleted; duplicated color and regex helpers were consolidated into
`scripts/detector/shared/color.mjs` and `scripts/lib/regex.mjs`; and the reference guides were
rewritten for the Claude Code harness. Do not try to re-vendor: it would revert these changes.
The provenance/attribution table above is retained only to meet the Apache-2.0/MIT notice terms.

Cross-reference hazard (still live): the craft guides link each other and the registers. `polish`
is the convergence hub (12 inbound), `craft` and `shape` fan out, and `typeset / brand / bolder`
form a knot. Do not remove guides piecemeal; you will orphan links and router entries.
